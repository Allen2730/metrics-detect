# 基于 AI 的 Prometheus 无效指标统计与智能识别分析工具设计与实现

## 适用说明

1. 作业形式：**居家独立完成，不限开发语言**（Go/Python/Java 均可）
2. 实现自由度：**允许接入大模型 API、本地开源小模型、AI 算法聚类分析、claude code、codex**等 AI 工具，不限制 AI 使用形式
3. 考察核心：**系统架构设计、Prometheus 指标与无效指标理解、TSDB 数据解析能力、工程模块化设计、AI 与可观测场景结合思路、云原生部署落地能力**
4. 完成周期：建议 3 天内提交（含代码 + 设计文档 + 架构说明）
5. 工具**必须完整遵循 Harness 核心思想与策略规范**，所有设计、实现、配置必须对齐 Harness 企业级云原生治理标准

## 一、业务背景

在云原生 Kubernetes 集群中，业务微服务大量接入 Prometheus 监控，随着服务迭代产生大量**无效指标（Invalid Metric）** 问题：

- 废弃下线指标、重复冗余指标、空标签 / 空值指标长期滞留
- 违规标签、无意义临时指标、日志泛滥生成的无效指标污染监控体系
- 导致 Prometheus 内存浪费、TSDB 存储膨胀、查询性能下降、告警误报 / 漏报
- 传统人工排查效率极低，缺少自动化统计、识别、风险评级、优化建议能力

要求你独立设计并实现一套 **AI 驱动的无效指标智能分析工具**，实现**指标采集→无效指标统计→智能识别→AI 归因→优化建议→报告输出**全链路能力。

---

## 二、整体要求

1. 可**模拟读取 Prometheus 原始指标文本**、也可对接真实 Prometheus HTTP 接口拉取指标元数据；
2. 支持解析 / 模拟 TSDB 时序标签结构，做**无效指标分类、统计、检测**；
3. 引入**AI 能力**做无效指标智能识别、风险分级、根因分析、给出可落地的监控优化建议；
4. 工具需具备良好模块化设计，支持容器化打包（Docker），可通过 Helm 适配 K8s 部署（方案设计即可，可不实际部署）；
5. 禁止只写 Demo 逻辑，要求具备**工程化结构、配置化管理、错误处理、日志输出**。

## 三、必须实现的核心功能

### 1. 指标数据接入与解析模块

- 支持两种接入方式任选其一实现：
  
  1. 读取本地 Prometheus 格式指标文本文件，解析 `指标名+多标签键值对`；
  2. 调用真实 Prometheus `/api/v1/label` `/api/v1/series` 接口拉取全量指标元数据；
- 标准化解析指标名、所有 Label Key/Label Value，结构化存入内存或简易文件。
  

### 2. 无效指标批量统计与分类能力

**先明确无效指标定义（必须覆盖）**：

废弃指标、重复指标、空名称 / 空标签 / 空值指标、违规字符标签指标、冗余无意义指标、无匹配服务的孤儿指标、高基数衍生无效指标

需完成以下维度统计：

1. 全量指标总数、有效指标数、**各类无效指标总数 / 占比**；
2. 按类型分类统计：废弃指标、重复指标、空值指标、违规标签指标数量；
3. 自动标记：**严重无效 / 一般无效 / 轻微无效** 三级风险分级；
4. 输出 Top20 高风险无效指标、Top20 违规标签维度。

### 3. AI 智能分析模块（核心必做，允许任意 AI 方案）

**任选一种 AI 方案实现即可**：

方案 A：调用公有大模型 API（GPT / 文心 / 通义等）

方案 B：本地开源小模型（LLaMA/Qwen 轻量化）做离线分析

方案 C：无大模型，用**聚类 / 特征工程 AI 算法**做无效指标特征聚类、异常识别

AI 需要输出能力：

1. 对识别出的无效指标做**风险等级评定**；
2. 智能分析**无效指标产生根因**（服务下线未清理、业务滥用标签、日志转指标失控、架构迭代遗留、配置错误等）；
3. 给出**可落地优化建议**：指标清理、重复指标合并、标签规范整改、Prometheus relabel 规则配置、TSDB 存储优化、监控治理规范；
4. 对批量指标生成**整体监控治理评估报告**。

### 4. 结果输出与报表能力

- 控制台结构化打印统计结果、无效指标风险列表；
- 支持导出 Markdown/Excel 分析报表；
- 记录分析日志，便于后续复盘。

## 四、可选加分功能（自愿实现）

1. 模拟 Prometheus TSDB 索引结构分析，识别无效指标对存储膨胀的影响；
2. 支持历史对比分析：两次扫描对比无效指标变化趋势；
3. 内置 Prometheus Relabel 规则自动生成，直接用于清理无效指标；
4. 简单命令行交互模式，支持手动指定指标、标签单独分析；
5. 支持指标有效性自动校验，对接 CI/CD 做准入拦截。
